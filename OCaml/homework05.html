CIS-3030 Homework #5

Read my OCaml tutorial. In the text read Chapter 15 on functional programming. You can skip sections 15.5, 15.6, and 15.8.

The questions below ask you to write several functions in OCaml. You do not need to write complete programs; the functions alone are enough. To test your code, you can type it into the OCaml top level and then execute it interactively. To simplify editing you can enter your code into a .ml file and use the #use directive in the OCaml top level to load that file. For example "#use myfuns.ml;;" will load the code in myfuns.ml into the top level. You do not need to separately compile the .ml file.

  1. The Fibonacci sequence is 0, 0, 1, 1, 2, 3, 5, 8, 13, ... It is defined as follows: F(0) = 0, F(1) = 1, F(n) = F(n-1) + F(n-2). In other words, each item in the sequence is the sum of the previous two items. Write a function that computes F(n) for any n ≥ 0. Note that the Fibonacci sequence grows exponentially. A simple recusive implementation is very slow when n is large. Don't worry about that.

  2. Write two versions of a function that computes the product of all the values in a list. For example, given the list [2; 2; 4; 4] the function should return 2*2*4*4 = 64. The first version should use the library List.fold_left function. The second version should not use fold_left but instead iterate down the list using a recursive helper function (that you'll need to write). The helper function should take two parameters: a "product so far" and the remaining list to process. For example given 4 and [5; 6], the helper function should return 4*5*6 = 120. The helper function should use pattern matching to break the given list into a head element and a tail list, and then call itself recursively to process the tail appropriately.

  3. (Tricky) Write a function unzip that takes a list of pairs, and returns a pair of lists where each list in the return value is composed of the corresponding elements from the input list. For example, given [(1, 2); (3, 4); (5, 6)], unzip should return ([1; 3; 5], [2; 4; 6]). Hint: Use pattern matching to break the input list into head and tail sections. Apply unzip to the tail in a nested let... in expression.

  4. (Tricky) Write a function zip that does the opposite from unzip above. For example, given a pair of lists ([1; 3; 5], [2; 4; 6]), zip should return the list of pairs [(1, 2); (3, 4); (5, 6)]. If the given lists are of unequal length, zip should only process elements that are matched, ignoring the "extra" elements in the longer list.
